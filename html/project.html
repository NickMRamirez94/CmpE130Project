Brute Force

This will always give the optimal answer. HOWEVER, be careful because this is VERY SLOW! Not applicable when trying to compute the path between 20 or more cities. This runs in O(n!), which is very slow.

To try out this Algorithm, please enter the number of cities you would like to calculate for.


Nearest Neighbor

This will rarely give the optimal answer (especially for a lot of cities). HOWEVER, this is much faster than Brute Force. This will run in O(n^2). The Algorithm itself is also very simple and works in the following way: Look at all current cities neighbors. Choose city that has the shortest path from current one AND that has not already been visited yet. Do the same for all the remainging cities until all cities have been visited and then head back to starting place.

To try out this Algorithm, please enter the number of cities you would like to calculate for.
